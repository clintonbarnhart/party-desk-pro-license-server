# Contributing

Development should be performed on feature branches created from `develop`.

## Workflow

1. Create `feature/<short-name>` from `develop`.
2. Keep each commit focused and descriptive.
3. Run PHP syntax checks before opening a pull request.
4. Open the pull request into `develop`.
5. Merge reviewed release changes from `develop` into `main`.

Never commit live Square access tokens, webhook signature keys, customer data, or WordPress secrets.
